## 术语速查

### 🧠 AI 核心基础概念

| 术语 | 一句话解释 |
|------|-----------|
| 人工智能 (AI) | 通过计算机程序模拟和实现人类智能的技术领域 |
| 机器学习 (ML) | 让计算机从数据中自动学习、无需明确编程的 AI 子领域 |
| 深度学习 (DL) | 使用多层神经网络处理复杂模式的 ML 分支 |
| 大语言模型 (LLM) | 基于海量文本预训练的大型神经网络，如 GPT-4、Claude |
| 生成式 AI | 专注于创造新内容（文本、图像、代码）的 AI 系统 |
| 通用人工智能 (AGI) | 具备与人类同等水平通用智能的终极目标 |

> 详见 [[wiki/concepts/AI 核心概念]]

### 🤖 AI Agent 体系

| 术语 | 一句话解释 |
|------|-----------|
| AI Agent | 自主感知环境、决策并执行动作的智能系统 |
| Agentic AI | 多 Agent 协作 + 动态任务分解 + 持久记忆的高级范式 |
| Agentic Workflow | 指导 Agent 协作完成复杂任务的结构化框架 |
| 多智能体系统 | 多个专长 Agent 通过交互协作解决复杂问题 |
| 自主智能体 | 具完整"感知→规划→行动→学习→适应"闭环的高度自主 Agent |
| 反应型智能体 | 仅基于当前感知和预设规则反应的简单 Agent |
| 学习型智能体 | 能从经验中学习并优化行为策略的 Agent |

> 详见 [[wiki/concepts/AI Agent]]

### 🔧 关键技术术语

| 术语 | 一句话解释 |
|------|-----------|
| 提示词工程 (Prompt Engineering) | 设计和优化输入指令以引导模型生成期望输出 |
| 上下文工程 (Context Engineering) | 系统性设计管理 AI 运行所需情境信息的工程实践 |
| 上下文窗口 (Context Window) | AI 模型一次能处理的最大 token 数 |
| 检索增强生成 (RAG) | 先从外部知识库检索再生成答案，减少幻觉 — 见 [[wiki/concepts/RAG]] |
| 函数调用 (Function Calling) | LLM 调用外部 API/函数的能力，使 AI 从"说"进化到"做" |
| 思维链 (Chain of Thought) | 引导模型展示逐步推理过程以提升复杂问题解决能力 |
| 计算机视觉 | 使计算机理解和处理图像/视频信息 |
| 强化学习 (RL) | 通过与环境交互、根据奖励信号学习最优策略 |
| 知识图谱 (KG) | 用图结构表示实体间关系的知识库 |

> 详见 [[wiki/concepts/AI 关键技术]]

### 🛠️ 开发框架与工具

| 术语 | 一句话解释 |
|------|-----------|
| LangChain | 流行 LLM 应用框架，提供模型、提示词、记忆、工具调用等模块化组件 |
| CrewAI | 模拟人类团队协作的多智能体框架，支持角色定义和结构化协同 |
| AutoGen | 微软推出的对话式多智能体框架，支持 Agent 间自然语言协商协作 |

> 详见 [[wiki/entities/AI Agent 开发框架]]

### 🔌 核心协议与架构

| 术语 | 一句话解释 |
|------|-----------|
| 模型上下文协议 (MCP) | AI 应用与外部工具/数据源"即插即用"连接的开放标准 |
| 智能体间通信协议 (A2A) | 定义不同 AI Agent 间直接协作通信的通用协议 |

> 详见 [[wiki/concepts/AI 协议与架构]]

### 💼 应用与场景

| 术语 | 一句话解释 |
|------|-----------|
| Copilot（副驾驶） | 人机协作模式，AI 辅助初稿、人类最终确认 |
| 数字员工 | AI Agent 在企业应用中像员工一样自主完成复杂业务流程 |
| Vibe Coding | 开发者用自然语言描述需求，AI 自动完成代码实现的编程范式 |
