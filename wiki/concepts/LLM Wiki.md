# LLM Wiki

> 一种由 LLM 持续维护、结构化互联的持久化知识库模式。

## 定义

LLM Wiki 是一种个人知识管理范式：**LLM 持续构建和维护一个结构化的 Wiki**，而不是在每次查询时从原始文档中检索片段。知识被编译一次后持续保鲜，Wiki 随着每个新源和每次查询不断变厚。

核心区别：LLM Wiki 是**持久化的、可复利的制品**（persistent, compounding artifact），而传统 RAG 每次都在重新发现知识。

## 核心思想

### 不是 RAG

RAG（检索增强生成）的工作方式：上传文档 → 查询时检索相关片段 → LLM 拼凑答案。每次查询从零开始，知识不积累，跨文档的综合依赖运气。

LLM Wiki 的工作方式：LLM **预先**通读源文档，提取关键信息，整合进现有 Wiki —— 更新实体页、修订概念综述、标记新旧矛盾。查询时，交叉引用已存在，矛盾已标记，综述已反映全部已摄入内容。

### 三个角色

| 角色 | 职责 |
|------|------|
| **人类** | 策展源、提问、思考含义、指导方向 |
| **LLM** | 摘要、交叉引用、归档、维护一致性 — 所有苦力活 |
| **Wiki** | 持久化知识制品，持续增长的代码库 |

### 为什么有效

人类放弃 Wiki 的原因：维护负担增长快于价值。更新交叉引用、保持摘要时效、维护一致性 —— 这些簿记工作枯燥且容易遗忘。

LLM 不无聊、不遗漏交叉引用、一次能触及 15 个页面。维护成本趋近于零，Wiki 持续保鲜。

## 三层架构

```
源文档 (raw/)     ← 不可变，人类提供
Wiki (wiki/)      ← LLM 全权维护
操作手册 (CLAUDE.md) ← 人类与 LLM 共同演化
```

## 三种操作

| 操作 | 触发 | LLM 做什么 |
|------|------|-----------|
| **Ingest** | "ingest" / "处理" / "摄入" | 读源 → 讨论 → 写摘要 → 更新索引 → 更新/创建关联页 → 记日志 |
| **Query** | 自然语言提问 | 读索引 → 定位相关页 → 引用合成答案 → 可选存回 Wiki |
| **Lint** | "lint" / "检查" | 扫矛盾 → 扫过期 → 找孤立页 → 找知识缺口 → 报告 |

## 关键文件

| 文件 | 用途 |
|------|------|
| `index.md` | 内容目录，每次摄入更新，查询时先读它 |
| `log.md` | 时间日志，追加模式，`grep "^## \["` 可解析 |
| `_templates/` | 页面模板，定义每种页面类型的结构 |
