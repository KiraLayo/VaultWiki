# LLM Wiki vs RAG

## 对比维度

| 维度 | [[wiki/concepts/LLM Wiki]] | [[wiki/concepts/RAG]] |
|------|---------------------------|----------------------|
| **知识积累** | ✅ 持续积累，Wiki 越来越厚 | ❌ 不积累，每次查询从零开始 |
| **查询方式** | 先读索引 → 精读相关页 → 引用合成 | 检索片段 → 注入上下文 → 生成答案 |
| **跨文档综合** | ✅ 交叉引用已预建，综述已综合 | ⚠️ 依赖检索覆盖，可能遗漏关键信息 |
| **一致性** | ✅ LLM 主动维护，矛盾被标记 | ❌ 不追踪矛盾 |
| **维护成本** | 人类近乎零（LLM 全包） | 零（因为不维护） |
| **可发现性** | 索引页 + 图视图 + wikilinks 导航 | 仅靠检索分数排序 |
| **基础设施** | 仅需 Obsidian + LLM Agent | 需要向量数据库 + 检索管线 |
| **适合规模** | 中等（~100 源，~几百页），可扩展 | 大量文档（成千上万） |
| **版本控制** | Git 免费获得 | 需自建 |
| **上手难度** | 需要搭建 Wiki 骨架 | 上传即用 |

## 选择建议

- **选 LLM Wiki**：你在一个领域持续深耕，需要知识的复利效应，愿意花时间策展源和指导 LLM
- **选 RAG**：文档量巨大（成千上万），只需要快速问答，不需要持久化知识结构
- **两者可共存**：RAG 做快速检索和发现，LLM Wiki 做深度整理和积累
