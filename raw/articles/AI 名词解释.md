# AI 名词解释

好的，根据提供的参考资料，我为您整理了AI领域的相关名词，并按类别进行了梳理，方便您理解。

### AI 核心基础概念

| **术语**                                             | **解释**                                                                    |
| -------------------------------------------------- | ------------------------------------------------------------------------- |
| **人工智能 (Artificial Intelligence, AI)**             | 通过计算机程序模拟和实现人类智能的技术领域。核心目标是让机器能自主地感知环境、做出决策并与环境交互。                        |
| **机器学习 (Machine Learning, ML)**                    | 人工智能的子领域，研究如何让计算机从数据中自动学习和改进性能，而无需明确编程。                                   |
| **深度学习 (Deep Learning, DL)**                       | 机器学习的一个分支，使用多层神经网络（如Transformer）来处理复杂模式，是当前大语言模型的核心技术。                    |
| **大语言模型 (Large Language Model, LLM)**              | 基于海量文本数据预训练的大型神经网络模型，具备强大的语言理解、生成和推理能力，如GPT-4、Claude等。常被视为AI Agent的“大脑”。  |
| **生成式AI (Generative AI)**                          | 专注于创造新内容（如文本、图像、代码）的AI系统。它被视为AI Agent的前身，主要关注内容生成，缺乏主动执行任务的能力。            |
| **通用人工智能 (Artificial General Intelligence, AGI)**  | 人工智能研究的终极目标，指具备与人类同等水平通用智能的系统，能理解、学习并应用知识解决各种问题。                          |

### AI Agent（智能体）相关

| **术语**                            | **解释**                                                                                              |
| --------------------------------- | --------------------------------------------------------------------------------------------------- |
| **AI Agent (AI智能体)**              | 一种能自主感知环境、进行决策并执行动作以实现特定目标的智能系统。它通常由“大模型 (LLM) + 规划 (Planning) + 记忆 (Memory) + 工具使用 (Tool Use)”构成。  |
| **Agentic AI (代理型人工智能)**          | 代表一种更高级的范式，强调由多个专业AI Agent组成系统，通过协作、动态任务分解和持久化记忆来实现更复杂的目标。它是一种 **“团队协作”** 模式。                       |
| **Agentic Workflow (智能体工作流)**     | 一个结构化框架，用于指导单个或多个AI Agent如何协作完成复杂任务。它定义了Agent行动的**具体步骤和方法**。                                        |
| **多智能体系统 (Multi-Agent System)**   | 由多个相互交互的AI Agent组成的系统，每个Agent拥有专长，通过协作解决单个Agent难以处理的复杂问题。例如CrewAI、AutoGen。                          |
| **自主智能体 (Autonomous Agent)**      | 具有高度自主性的AI Agent，具备完整的“感知 → 规划 → 行动 → 学习 → 适应”闭环，无需人类持续指导即可完成任务，如AutoGPT。                           |
| **反应型智能体 (Simple Reflex Agent)**  | 仅基于当前感知和预设的“条件-动作”规则做出反应的简单Agent，不考虑历史信息。                                                           |
| **学习型智能体 (Learning Agent)**       | 具备从过往经验中学习和优化自身行为策略能力的Agent，如通过强化学习训练的自动驾驶系统。                                                       |

### AI 关键技术术语

| **术语**                                            | **解释**                                                       |
| ------------------------------------------------- | ------------------------------------------------------------ |
| **提示词工程 (Prompt Engineering)**                    | 设计和优化输入给AI模型的指令（提示词），以引导模型生成期望输出的技术。                         |
| **上下文工程 (Context Engineering)**                   | 系统性设计和管理AI运行所需的所有情境信息（如用户目标、历史、业务规则）的工程实践，是构建高质量Agent应用的核心。  |
| **上下文窗口 (Context Window)**                        | AI模型一次能处理的最大token数，决定了它能“记住”的信息量。                            |
| **检索增强生成 (Retrieval-Augmented Generation, RAG)**  | 一种技术，让AI模型在生成回答前，先从外部知识库检索相关信息，以提升答案的准确性和时效性，减少“幻觉”。         |
| **函数调用 (Function Calling)**                       | LLM能调用外部API或函数的能力，使AI从“只能说”进化到“能做事”，如查天气、订机票。                |
| **思维链 (Chain of Thought, CoT)**                   | 一种引导模型进行逐步推理的技术，让模型展示其思考过程，以提升解决复杂问题的能力。                     |
| **计算机视觉 (Computer Vision)**                       | 使计算机能够从图像和视频中理解和处理视觉信息的技术，让AI Agent能“看见”。                    |
| **强化学习 (Reinforcement Learning, RL)**             | 一种机器学习方法，智能体通过与环境的试错交互，根据奖励信号学习最优行为策略。                       |
| **知识图谱 (Knowledge Graph, KG)**                    | 一种用图结构表示实体及其之间关系的知识库，为AI Agent提供结构化的常识和背景知识，支持更复杂的推理。        |

### 开发框架与工具

| **术语**        | **解释**                                                 |
| ------------- | ------------------------------------------------------ |
| **LangChain** | 一个流行的AI Agent开发框架，提供了构建LLM应用所需的模块化组件，如模型、提示词、记忆和工具调用。  |
| **CrewAI**    | 一个专注于模拟人类团队协作的多智能体框架，允许开发者定义角色和任务，实现结构化协同。             |
| **AutoGen**   | 由微软推出的对话式多智能体编程框架，支持Agent间通过自然语言对话进行协商和协作。             |

### 核心协议与架构

| **术语**                                     | **解释**                                                 |
| ------------------------------------------ | ------------------------------------------------------ |
| **模型上下文协议 (Model Context Protocol, MCP)**  | 一种开放标准，旨在实现AI模型与外部工具、数据源之间的“即插即用”式连接，类似于AI应用的“USB接口”。  |
| **智能体间通信协议 (Agent-to-Agent, A2A)**         | 一种新兴协议，用于定义不同AI Agent之间进行直接协作和通信的通用语言，旨在实现跨系统的大规模协作。   |

### 应用与场景

| **术语**                      | **解释**                                            |
| --------------------------- | ------------------------------------------------- |
| **Copilot (副驾驶)**           | 一种人机协作模式，AI辅助人类完成工作初稿，人类进行最终确认和调整。与ChatGPT代表L3级别。 |
| **数字员工 (Digital Employee)** | AI Agent在企业应用中的形象化称呼，代表其能像员工一样自主完成复杂业务流程。         |
| **Vibe Coding**             | 一种新型编程范式，开发者只需用自然语言描述需求，AI即可自动完成代码实现和工程交付。        |
